# 我应该做什么
来自Qwen
1. 聊“文献阅读笔记” (输入)
如果你实验还没做，或者代码还没跑，你一定在看文献吧？
怎么说： “老师，我最近读了这几篇核心文献，我发现他们用的方法主要是A，但是我觉得在这个特定场景下可能有局限性。您觉得我理解得对吗？”
价值： 证明你在思考，你在通过文献寻找切入点。
2. 聊“实验/研究设计” (规划)
还没动手，但脑子里应该有计划了吧？
怎么说： “老师，接下来我打算做X实验。我目前的计划是控制变量Y，观测指标Z。但我担心会有W干扰，您觉得这个设计逻辑通顺吗？”
价值： 这是最高效的交流。在行动前获得老师的确认，比你做完被否决要划算得多。
3. 聊“遇到的具体困难” (卡点)
做得不多，是不是因为卡住了？
怎么说： “老师，我尝试复现这个代码/搭建这个环境，但在配置A这里卡了两天了，我查了网上的方案1和2都没解决，您有什么建议吗？”
价值： 展示你的努力，并寻求技术支援。
4. 聊“失败的尝试” (避坑)
哪怕只做了一次实验失败了，这也是极好的交流素材。
怎么说： “老师，我试了一下方法A，结果数据完全对不上。我分析原因可能是B，所以我打算下一步尝试C。您觉得我的分析合理吗？”
价值： 科研中，排除错误路径也是贡献。 老师能帮你判断是操作失误还是方向错误。
